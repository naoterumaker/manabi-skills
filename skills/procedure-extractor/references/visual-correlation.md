# Visual Correlation ガイド

トランスクリプトのアクション記述とスクリーンショットを紐付けるための詳細ガイド。

---

## 1. アクション動詞パターン（日本語）

トランスクリプトから手順を検出する際に使うパターン。

### クリック・タップ系

| パターン | 例文 |
|---------|------|
| クリック | 「ここをクリックします」 |
| タップ | 「タップしてください」 |
| 押す / 押して | 「このボタンを押して」 |
| ポチッと | 「ポチッと押すだけ」 |
| ダブルクリック | 「ダブルクリックで開きます」 |

### 入力・記述系

| パターン | 例文 |
|---------|------|
| 入力 | 「ここに入力してください」 |
| 入れて / 入れる | 「キーワードを入れて」 |
| 打って / 打つ | 「URLを打ってください」 |
| 書いて / 書く | 「名前を書いてください」 |
| 貼り付け / ペースト | 「コピーして貼り付けて」 |
| コピー | 「このURLをコピーします」 |

### 選択・チェック系

| パターン | 例文 |
|---------|------|
| 選択 / 選んで | 「こちらを選択してください」 |
| チェック | 「チェックを入れて」 |
| オン / オフ | 「ここをオンにします」 |
| 切り替え / トグル | 「トグルを切り替えて」 |
| ドロップダウン | 「ドロップダウンから選んで」 |

### 移動・遷移系

| パターン | 例文 |
|---------|------|
| 開く / 開いて | 「設定画面を開いてください」 |
| アクセス | 「このURLにアクセスします」 |
| 移動 / 行って | 「次のページに移動します」 |
| 飛んで / 飛ぶ | 「リンクに飛んでください」 |
| 遷移 | 「画面が遷移します」 |
| 戻って / 戻る | 「前の画面に戻ります」 |
| スクロール | 「下にスクロールして」 |

### 設定・変更系

| パターン | 例文 |
|---------|------|
| 設定 | 「以下のように設定します」 |
| 変更 | 「名前を変更してください」 |
| 登録 | 「情報を登録します」 |
| 保存 | 「保存ボタンを押します」 |
| 削除 | 「不要なものを削除して」 |
| 更新 | 「ページを更新します」 |

### 指示語+動詞（高確率で画面操作）

| パターン | 理由 |
|---------|------|
| 「ここを〜」 | 画面上の特定箇所を指している |
| 「この画面で〜」 | 現在表示中の画面を参照 |
| 「こちらの〜」 | 画面上の要素を指示 |
| 「右上の〜」「左下の〜」 | 画面位置の指定 |
| 「赤い（青い）ボタン〜」 | UI要素の視覚的特徴 |

## 2. タイムスタンプの推定

### transcript_ts.json がある場合

`transcript_ts.json` にはセグメントごとのタイムスタンプがある:

```json
[
  { "start": 120.5, "end": 125.3, "text": "ここをクリックしてください" },
  { "start": 125.3, "end": 130.1, "text": "次に検索欄に入力します" }
]
```

アクション動詞が含まれるセグメントの `start` を基準に、visual-index.json のフレームを検索する。

### transcript_ts.json がない場合

位置ベースの推定を使う:

```
推定タイムスタンプ = (アクション出現位置 / トランスクリプト全文字数) × チャプター総時間
```

チャプター総時間は manifest.json の duration、または visual-index.json の最後のフレームのタイムスタンプから推定する。

精度は低いため、候補を広めに（±15秒）取り、画像内容で最終判断する。

## 3. フレームとのマッチング

### visual-index.json のフレーム構造

```json
{
  "frame": "frame_012.jpg",
  "timestamp": 123.4,
  "type": "browser_screenshot",
  "description": "noteの検索画面"
}
```

### マッチング手順

1. **タイムスタンプで候補を絞る**: アクションのタイムスタンプ ±5秒（ts.jsonなしなら±15秒）のフレームを候補にする
2. **typeでフィルタ**: `browser_screenshot` または `terminal_screenshot` を優先。`talking_head` や `slide` は手順には使わない
3. **画像を実際に読む**: Read ツールで候補フレームを表示し、内容を確認する
4. **照合判断**: トランスクリプトの操作内容と画像内容が一致するか判断する

### 照合の判断基準

| 一致と判断する場合 | 不一致と判断する場合 |
|------------------|-------------------|
| 「noteの検索画面」→ 画像にnoteのUIが映っている | 「noteの検索画面」→ 画像はスライド資料 |
| 「入力して」→ テキスト入力欄がアクティブ | 「入力して」→ 結果一覧画面 |
| 「クリック」→ 操作前の画面（これからクリックする対象が見える） | 「クリック」→ 全く別の画面 |

**原則**: screenshot_ref は「操作を行う直前の画面」または「操作中の画面」のフレームを指す。「操作後の結果画面」はそのステップではなく、次のステップや outcome の参考にする。

## 4. エッジケース

### 4.1 複数ステップが同一スクリーンショットを共有

同じ画面内で複数の操作を行う場合（例: フォームの複数フィールドに入力）、同じ screenshot_ref を複数ステップで参照してよい。

```json
{ "step": 1, "action": "名前欄に入力", "screenshot_ref": "frame_020.jpg" },
{ "step": 2, "action": "メール欄に入力", "screenshot_ref": "frame_020.jpg" },
{ "step": 3, "action": "送信ボタンをクリック", "screenshot_ref": "frame_020.jpg" }
```

### 4.2 スクリーンショットがないステップ

口頭での説明のみで画面操作がない、またはカメラが講師を映しているときは `null` にする。

```json
{ "step": 2, "action": "別タブでGmailを開く", "screenshot_ref": null }
```

無理にスクショを付けない。存在しないファイルを参照するよりは null のほうが安全。

### 4.3 手順が複数チャプターにまたがる

1つの操作手順が2つのチャプターにまたがる場合:
- 各チャプターで独立した procedure として記録する
- 前チャプターの procedure を prerequisites に含める
- title に「(続き)」等を付けて関連を示す

```json
{
  "id": "p07-01",
  "title": "LINE公式アカウントの設定（続き: メッセージ配信設定）",
  "prerequisites": ["p06-03"],
  ...
}
```

注意: prerequisites の形式は concept ID (`c{xx}-{yy}`) だが、procedure 間の依存は title と tags で表現する。スキーマ上 prerequisites は concept への参照に限定されている。

### 4.4 スクリーンショットの画質が悪い / 判読不能

画像を読んでも内容が判別できない場合は `screenshot_ref: null` にし、action の記述をより詳細にする。

### 4.5 同一タイムスタンプに複数フレーム

visual-index.json で同タイムスタンプに複数フレームがある場合は、type が `browser_screenshot` / `terminal_screenshot` のものを優先し、さらに description がトランスクリプトの文脈に近いものを選ぶ。
